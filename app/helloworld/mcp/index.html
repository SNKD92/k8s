How This MCP Chat Is Deployed and Works

This page and its chat backend are deployed through the same platform pipeline: container build, Kubernetes rollout, and secure in-cluster routing to Ollama.

Deployment Path

  1. Code changes are pushed to main.
  2. GitHub Actions builds and pushes a new app image.
  3. Kubernetes deployment is updated and rolled out.
  4. LLM manifests run Ollama on the dedicated llm-pool.
  5. Ingress serves the updated MCP page on /mcp.

Runtime Request Path

  1. Your browser loads the static MCP UI from NGINX.
  2. Chat calls /ollama/api/chat on the same domain.
  3. NGINX proxies that path to ollama.llm.svc inside the cluster.
  4. Ollama runs the model and returns the response through the same path.
  5. The UI renders the model response in the chat stream.
