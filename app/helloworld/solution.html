Infrastructure

Plain words: We create cloud infrastructure that can reliably run the website.

Technical details
  • Terraform provisions GKE cluster small-gke in us-central1-a.
  • Node config uses e2-small and preemptible instances to reduce cost.
  • Provider settings and outputs are managed in main.tf, variables.tf, and outputs.tf.

Links: gke.tf main.tf outputs.tf

Application

Plain words: The app is a lightweight static website served by NGINX in a container.

Technical details
  • Docker image is based on nginx:alpine.
  • index.html is the home page, and solution.html is the architecture guide.
  • Container build-time metadata is baked into both pages at image build.

Links: Dockerfile index.html app folder

Kubernetes Deployment

Plain words: Kubernetes keeps the app running and exposes it to internet users.

Technical details
  • Deployment runs 3 replicas for availability.
  • Service exposes internal port 80 via ClusterIP.
  • Ingress routes domain traffic to the service and attaches TLS cert secret.

Links: deployment.yaml service.yaml ingress.yaml

cert-manager and HTTPS

Plain words: SSL certificates are issued automatically so the site is always on HTTPS.

Technical details
  • ClusterIssuer uses Let's Encrypt ACME production endpoint.
  • DNS-01 challenge is solved through Cloudflare using cloudflare-api-token-secret.
  • Certificate resource requests cert for deniskachar.com into deniskachar-com-tls.

Links: clusterissuer.yaml certificate.yaml cloudflare-secret.yaml

CI/CD Pipeline

Plain words: Every push to main can build, publish, and deploy the latest version automatically.

Technical details
  • GitHub Actions workflow builds Docker image with version tag and latest.
  • Image is pushed to Docker Hub with repository secrets.
  • Workflow gets GKE credentials and updates deployment image using kubectl set image.

Links: deploy.yml deployment image target

Identity and Secrets

Plain words: Access is controlled using short-lived identity tokens instead of long-lived keys.

Technical details
  • Terraform creates Workload Identity Pool and Provider for GitHub OIDC federation.
  • GitHub workflow requests OIDC token and impersonates service account for deployment actions.
  • Cloudflare and Docker credentials are stored as secrets outside Git source code.

Links: sa.tf workflow auth step

End-to-End Delivery Flow

Plain words: This is the sequence from infrastructure to live production updates.

Technical details
  1. Terraform provisions GKE and IAM/OIDC resources.
  2. cert-manager and certificate objects are applied.
  3. App deployment, service, and ingress are created in Kubernetes.
  4. Push to main triggers build and deploy pipeline.
  5. Ingress serves the new version over HTTPS on deniskachar.com.

Links: deployment guide k8s manifests
