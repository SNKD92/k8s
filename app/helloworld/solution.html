Tech: Let's Encrypt ACME + Cloudflare DNS challenge, stored in deniskachar-com-tls.

Release Flow (Real Sequence)

This is the actual order used in this repo and deployment pipeline.

1

Infra baseline

terraform apply

Used when cluster/IAM config changes.

2

TLS and DNS challenge

kubectl apply -f k8s/cert-manager/

Certificate is issued and stored as ingress secret.

3

App manifests live

kubectl apply -f k8s/app/

Deployment + service + ingress become active.

4

Delivery trigger

git push origin main

Push starts .github/workflows/deploy.yml.

5

Rolling update

kubectl rollout status deployment/helloworld-deployment

Pipeline completes only after rollout is healthy.
