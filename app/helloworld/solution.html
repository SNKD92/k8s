Infrastructure (Terraform / GKE)

  • infra/terraform/gke.tf provisions cluster small-gke in us-central1-a.
  • Current node profile is cost-focused: e2-small, preemptible, with IP allocation enabled.
  • main.tf, variables.tf, and outputs.tf keep core project/cluster config together.

App Layer (Container + Runtime)

  • app/helloworld/Dockerfile builds a small nginx:alpine image.
  • NGINX serves static content from /usr/share/nginx/html with UTF-8 enabled.
  • index.html is the home page and solution.html is the technical overview page.

Kubernetes Deployment

  • k8s/app/deployment.yaml runs 3 replicas of the app container.
  • k8s/app/service.yaml exposes pods internally on port 80 via ClusterIP.
  • k8s/app/ingress.yaml routes deniskachar.com to the service with TLS.

cert-manager + TLS

  • clusterissuer.yaml defines letsencrypt-cloudflare using Let's Encrypt ACME.
  • Domain validation uses Cloudflare DNS-01 and secret cloudflare-api-token-secret.
  • certificate.yaml requests the cert and stores it in deniskachar-com-tls.

CI/CD Pipeline (GitHub Actions)

  • .github/workflows/deploy.yml runs on pushes to main.
  • It builds the image, pushes timestamp + latest tags to Docker Hub, then deploys to GKE.
  • Deployment step updates the image in Kubernetes and waits for rollout status.

Identity and Secret Management

  • infra/terraform/sa.tf creates GitHub Actions service account + Workload Identity federation.
  • GCP auth uses OIDC, so there is no static service-account key in the repo.
  • Cloudflare and Docker credentials are managed through secrets, not committed files.

End-to-End Deployment Flow

  1. Provision infrastructure with Terraform (GKE + IAM/WIF resources).
  2. Install cert-manager and apply issuer/certificate manifests.
  3. Deploy app manifests (deployment, service, ingress).
  4. Push to main to trigger build-and-deploy workflow.
  5. Ingress serves the updated app over HTTPS on deniskachar.com.
