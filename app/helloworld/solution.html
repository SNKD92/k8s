Infrastructure (Terraform / GKE)

  • infra/terraform/gke.tf creates a GKE cluster small-gke in us-central1-a.
  • Node profile is optimized for low cost: e2-small, preemptible nodes, and IP aliasing enabled.
  • infra/terraform/main.tf configures Google provider with project, region, and zone variables.
  • infra/terraform/outputs.tf exports cluster name/zone and GitHub Actions service account email.
Terraform Google Cloud GKE

App Layer (Container + Runtime)

  • app/helloworld/Dockerfile builds a lightweight NGINX container from nginx:alpine.
  • Static pages are served from /usr/share/nginx/html with UTF-8 enabled.
  • app/helloworld/index.html is the landing page.
  • app/helloworld/solution.html (this page) documents architecture and deployment flow.
Docker NGINX Static HTML

Kubernetes Deployment

  • k8s/app/deployment.yaml runs 3 replicas of snkd92/helloworld:latest.
  • k8s/app/service.yaml exposes pods internally via ClusterIP on port 80.
  • k8s/app/ingress.yaml routes external HTTPS traffic for deniskachar.com.
  • TLS secret used by ingress: deniskachar-com-tls.
Kubernetes Ingress Rolling Update

cert-manager + TLS

  • k8s/cert-manager/clusterissuer.yaml defines letsencrypt-cloudflare.
  • ACME server: Let's Encrypt production endpoint.
  • DNS-01 challenge is solved through Cloudflare API token secret: cloudflare-api-token-secret in namespace cert-manager.
  • k8s/cert-manager/certificate.yaml requests cert for deniskachar.com.
cert-manager Let's Encrypt Cloudflare DNS-01

CI/CD Pipeline (GitHub Actions)

  • .github/workflows/deploy.yml triggers on pushes to main.
  • Builds image with a timestamp tag and also updates latest.
  • Pushes to Docker Hub using repository secrets DOCKER_USERNAME and DOCKER_PASSWORD.
  • Authenticates to GCP through Workload Identity Federation (OIDC) without static SA key files.
  • Updates GKE deployment image and waits for rollout completion.
GitHub Actions OIDC Workload Identity

Identity and Secret Management

  • infra/terraform/sa.tf creates service account github-actions and WIF pool/provider.
  • Repository scope is restricted to SNKD92/k8s for federated identity.
  • Cloudflare token is restored from GCP Secret Manager into Kubernetes secret at deploy time.
  • Docker Hub password source of truth is GCP Secret Manager secret: projects/365212276900/secrets/githubactionssecret.
IAM Secret Manager Least Privilege

End-to-End Deployment Flow

  1. Terraform provisions GKE + GitHub OIDC federation resources.
  2. cert-manager is installed and configured with Cloudflare DNS challenge.
  3. Certificate for deniskachar.com is issued and stored in deniskachar-com-tls.
  4. GitHub Actions builds and pushes a new Docker image.
  5. Workflow updates Kubernetes deployment image and verifies rollout status.
  6. Ingress serves HTTPS traffic to the updated application.
